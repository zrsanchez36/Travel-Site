Programming Principials

SOLID

Single Responsibility: Breaking our code down into modules so each can focus on 1 responsibility each.
Open/Closed: Create code/modules to be able to add new functonality in the future without having to make changes directly. Extend modules by wrapping or anything, we should never modify it directly. Once a module is in use it's locked.
Liskov Substitution: Only extend modules when we are sure that the extension will keep it the same at heart.
Interface Segrgation: Our modules don't need to know about functionality that it doesn't use. Split our modules into smaller abstractions like interfaces that we can then compose to form the exact functionality that a module requires.
